Project architecture

How the system works in code, not in marketing slides.

This page reflects the current backend implementation: modular Flask application, persisted settings, protected admin flow, asynchronous processing and generated artifacts.

Open pipeline Inspect operations

System domain

Asynchronous article extraction and summarization.

The application ingests public article URLs, extracts meaningful content, summarizes the text through extractive or generative strategies and stores task history for later review.

Architecture shape

Modular monolith with layered boundaries.

Presentation routes live under Flask blueprints, application handlers orchestrate commands and queries, domain entities capture business state, and infrastructure adapters integrate database, auth, cache and the summarization pipeline.

Presentation

HTML and API entry points

Flask blueprints expose web pages, REST endpoints and auth flows while CSP nonce and security headers are applied centrally.

Application

Commands, queries and handlers

Task submission, runtime settings updates, authentication and cache management are coordinated through explicit handler classes.

Domain

Entities and events

Tasks, users and audit log entries live in the domain layer while submission, completion and failure events drive async processing and audit trails.

Infrastructure

Persistence and runtime services

SQLAlchemy repositories, Alembic migrations, Redis-backed cache, Celery dispatching and JWT secret rotation complete the production-facing edge.

Runtime flow

What happens after a user submits a URL

01

API validation

The request is normalized, rate limited and persisted as a task.

02

Event dispatch

A task submitted event dispatches asynchronous processing through Celery or local thread fallback.

03

Pipeline execution

Scraper, text processor, summarizer and file manager produce the final result and downloadable artifacts.

04

History and admin visibility

The frontend reads task status, protected statistics, persisted settings and audit-backed admin actions from the real API.
